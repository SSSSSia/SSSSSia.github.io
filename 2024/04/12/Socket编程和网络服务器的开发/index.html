2.1 套接字

2.1.1 套接字概念

Socket中文意思是“插座”，在Linux环境下，用表示进程间网络通信的特殊文件类型。本质为内核借助缓冲区形成的伪文件。

既然是文件，那么理所当然的，我们可以使用文件描述符引用套接字。Linux系统将其封装成文件的目的是为了统一接口，使得读写套接字和读写文件的操作一致。区别是文件主要应用于本地持久化数据的读与，而套接字多应用于网络进程间数据的传递。

在TCP/IP协议中，“IP地址+TCP或UDP端口号”唯一标识网络通讯中的一个进程。“IP地址+端口号”就对应一个socket,欲建立连接的两个进程各自有一个socket来标识，那么这两个socket组成的socket pair就唯一标识一个连接。因此可以用Socket来描述网络连接的一对一关系。

2.1.2 套接字与文件描述符

在套接字编程中，虽然套接字是成对出现的，每个套接字会关联一个文件描述符，但是这个文件描述符同时负责发送缓冲区和接收缓冲区。

具体来说，当建立一个TCP连接时，本地套接字和远程套接字会各自有一个发送缓冲区和接收缓冲区。本地套接字的发送缓冲区用于存储准备发送到远程套接字的数据，而接收缓冲区则用于存储从远程套接字接收到的数据。远程套接字也是如此。这四个缓冲区（两个本地，两个远程）是成对出现的，但是它们并不是通过同一个文件描述符来访问的，而是通过各自的套接字描述符进行操作。

因此，每个套接字都有自己的文件描述符，用于区分是读取数据还是发送数据。在进行套接字编程时，我们通过这个文件描述符来读取（对应接收缓冲区）和发送（对应发送缓冲区）数据，而不是使用不同的文件描述符来访问发送缓冲区和接收缓冲区

2.2.1 网络字节序

​ 在计算机世界里，有两种字节序：

  • 大端字节序 - 低地址高字节，高地址低字节
  • 小端字节序 - 低地址低字节，高地址高字节

​ 内存中的多字节数据相对于内存地址有大端和小端之分，盛盘文件中的多字节数据相对于文件中的偏移地址也有大端小端之分。网络数据流同样有大端小端之分，那么如何定义网络数据流的地址呢，发送主机通常将发送缓冲区中的数据按内存地址从低到高的顺序发出，接收主机把从网络上接到的字节依次保存在按收缓冲区中，也是按内存地址从低到高的顺序保存，因此，网络数据流的地址应这样规定:先发出的数据是低地址，后发出的数据是高地址。

TCP/IP 协议规定，网络数据流应采用大端字节序，即低地址高字节。

​ 例如端口号是 1001 (0x03e9)，由两个字节保存，则低地址是 0x03，高地址是 0xe9，也就是先发 0x03，再发 0xe9，这 16位在发送主机的缓冲区中也应该是低地址存0x03，高地址存 0xe9。但是，如果发送主机是小端字节序的，这 16 位被解释成 0xe903，而不是 1001。因此，发送主机把 1001 填到发送缓冲区之前需要做节序的转换。同样的，接收主机如果是小端字节序的，接到 16 位的源端口号也要做字节序的转换。如果主机是大端字节序的，发送和接收都不需要做转换。同理，32 位的 IP 地址也要考虑网络字节序和主机字节序的问题。

为使网络程序具有可移植性，使同样的C 代码在大端和小端计算机上编译后都能正常运行，可以调用以下库函数做网络字节序（大端）和主机字节序（大/小端）的转换。

补充知识： 关于网卡、网口和 IP地址
  1. 网卡是实现物理连接和数据传输的硬件： 网卡是计算机中的一个硬件组件，它负责将计算机内部的数据转换为可以在网络上传输的信号，并将网络上的信号转换回计算机内部的数据。网卡通过电缆（如以太网电缆）或无线信号与网络连接，实现数据的发送和接收。
  2. 网口是网卡的物理接口： 网口是网卡上的一个物理连接点，用于插入网络电缆，从而将计算机连接到网络。例如，RJ-45 接口就是一种常见的以太网网口。
  3. IP 地址是一个逻辑地址，用于在网络层上标识和通信： IP 地址是一个分配给网络设备的地址，用于在网络层（如 Internet Protocol 层）上标识设备，并允许设备之间进行通信。IP 地址是一个逻辑概念，它不直接与硬件相关，而是用于网络层的数据包路由和寻址。
  4. 一个网卡可以有多个网口，每个网口可以分配一个或多个 IP 地址： 这句话说明了网卡和网口的灵活性。一个网卡可以具有多个物理网口，每个网口都可以独立地连接到网络，并且可以分配不同的 IP 地址。这种情况在实际中比较少见，因为大多数个人计算机的网卡只有一个网口。然而，在服务器和高性能网络设备中，可能会配置有多个网口，以提供更高的网络带宽或实现网络的冗余和分离。
  5. 以实现不同网络协议和数据流的分离： 这是指通过为不同的网口分配不同的 IP 地址，可以在网络中实现流量的分离。例如，一个网口可能用于公共互联网的连接，而另一个网口可能用于内部网络的连接。这样，不同的网络流量可以通过不同的网口和 IP 地址进行管理和路由，提高了网络的效率和安全性。

典型的服务器程序可以同时服务于多个客户端，当有客户端发起连接时，服务器调用的 accept() 返回并接受这个连接，如果有大量的客户端发起连接而服务器来不及处理，尚未 accept 的客户端就处于连接等待状态，listen() 声明 sockfd 处于监听状态，并且最多允许有 backlog 个客户端处于连接待状态，如果接收到更多的连接请求就忽略。listen() 成功返回0，失败返回-1。

2.3.4 accept 函数

sockfd：
socket 文件描述符

addr：
传出参数，返回链接客户端地址信息，含 IP 地址和端口号

addrlen：
传入传出参数（指-结果），传入sizeof(addr)大小，函数返回时返回真正接收到地址结构体的大小

返回值：
成功返回一个心得socket 文件描述符，用于和客户端通信，失败返回 -1，设置errno

​ 三方握手完成后，服务器调用 accept() 接受连接，如果服务器调用 accept() 时还没有客户端的连接请求，就阻塞等待直到有客户端连接上来。addr 是一个传出参数，accept() 返回时传出客户端的地址和端口号。addrlen 参数是一个传入传出参数(value-result argument)，传入的是调用者提供的缓冲区 addr 的长度以避免缓冲区溢出问题，传出的是写入的客户端地址结构体的实际长度 (有可能没有占满调用者提供的缓冲区)。如果给 addr 参数传 NULL，表示不关心客户端的地址.
